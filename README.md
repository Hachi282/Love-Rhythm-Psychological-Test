# 她的戀愛節奏心理測驗

一個部署在 GitHub Pages 的純前端互動測驗網站。  
使用者完成 10 題作答後，系統會根據三軸分數組合出 16 種人格結果，並提供：

- 結果標題與主分析
- 第二層個性拆解
- 推薦歌曲與外部試聽連結
- 雙人分享連結配對模式

## 專案亮點

### 1. 純前端、可直接部署

整個專案不依賴建置工具，也不需要後端。  
只要把檔案放到 GitHub Pages，就能直接上線。

### 2. 內容與邏輯拆分

把測驗拆成幾個獨立模組，方便修改與維護：

- `questions.js`：題庫與選項分數
- `results.json`：16 種人格結果主資料
- `result-details.js`：結果頁第二層文字
- `engine.js`：三軸分數計算與結果判定
- `app.js`：互動流程、畫面渲染、分享與配對邏輯

### 3. 雙人分享連結模式

專案有做一個不依賴資料庫的雙人玩法：

1. A 完成測驗
2. 系統把 A 的結果摘要與三軸分數編進網址參數 `?pair=...`
3. A 把連結傳給 B
4. B 完成測驗後，前端直接比較兩人的結果並輸出配對分析

優點：

- 不需要帳號系統
- 不需要後端儲存
- 適合 GitHub Pages 這種靜態網站

### 核心設計取捨

#### 為什麼不用後端？

這個專案的目標是快速部署、容易分享、維護成本低。  
因此優先採用純前端方案，把資料放在靜態檔案中，並以 URL 分享機制處理雙人模式。

#### 為什麼歌曲只做外部連結？

結果頁會提供 YouTube / Spotify 搜尋連結，但不直接放歌詞或自行託管音訊。  
這是為了降低著作權風險，也讓 GitHub Pages 版本維持簡潔。

## 檔案結構

- `index.html`：網站入口
- `app.js`：互動流程、結果頁渲染、雙人分享連結與配對分析
- `styles.css`：整體視覺與響應式排版
- `questions.js`：題庫
- `results.json`：16 種人格結果主分析
- `result-details.js`：結果頁第二層文案
- `engine.js`：三軸分數計算與結果判定

## 本機預覽

因為頁面會用 `fetch('./results.json')` 讀資料，請不要直接雙擊 `index.html` 用 `file://` 開啟。

可以在專案資料夾執行：

```bash
python -m http.server 8000
```

然後打開：

`http://localhost:8000`

## 可自訂項目

- 想改題目文字：編輯 `questions.js`
- 想改結果主分析或歌曲：編輯 `results.json`
- 想改第二層文字：編輯 `result-details.js`
- 想改計分規則：編輯 `engine.js`
- 想改雙人配對文案與分享邏輯：編輯 `app.js`
- 想改版型與視覺：編輯 `styles.css`
